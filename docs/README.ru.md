# CC-Viewer

Система мониторинга запросов Claude Code — перехватывает и визуализирует в реальном времени все API-запросы и ответы Claude Code (исходный текст, без сокращений). Позволяет разработчикам отслеживать свой контекст для удобного анализа и отладки в процессе Vibe Coding.
Последняя версия CC-Viewer также предоставляет решения для серверного веб-развёртывания и инструменты для мобильной разработки. Приглашаем использовать в своих проектах — в будущем будет доступно больше возможностей и поддержка облачного развёртывания.

## Использование

### Установка

```bash
npm install -g cc-viewer --registry=https://registry.npmjs.org
```

### Режим мониторинга (при запуске claude или claude --dangerously-skip-permissions автоматически запускается фоновый процесс для записи запросов)

```bash
ccv
```

### Режим программирования

== claude

```bash
ccv -c
```

== claude --dangerously-skip-permissions

```bash
ccv -d
```

После запуска режима программирования автоматически откроется веб-страница.

На веб-странице можно напрямую работать с claude, а также просматривать полные тела запросов и изменения кода.

И что ещё интереснее — вы даже можете программировать с мобильного устройства!

Команда автоматически определяет способ установки локального Claude Code (NPM или Native Install) и адаптируется к нему.

- **Установка через NPM**: автоматически внедряет перехватывающий скрипт в `cli.js` Claude Code.
- **Native Install**: автоматически обнаруживает бинарный файл `claude`, настраивает локальный прозрачный прокси и устанавливает Zsh Shell Hook для автоматической переадресации трафика.
- Рекомендуется использовать Claude Code, установленный через npm.

### Переопределение конфигурации (Configuration Override)

Если вам нужно использовать пользовательскую конечную точку API (например, корпоративный прокси), просто настройте `~/.claude/settings.json` или задайте переменную окружения `ANTHROPIC_BASE_URL`. `ccv` автоматически распознает и корректно перенаправит запросы.

### Тихий режим (Silent Mode)

По умолчанию `ccv` работает в тихом режиме при запуске `claude`, обеспечивая чистый вывод в терминале, идентичный нативному. Все логи перехватываются в фоне и доступны по адресу `http://localhost:7008`.

После настройки просто используйте команду `claude` как обычно. Откройте `http://localhost:7008` для просмотра интерфейса мониторинга.

### Устранение неполадок (Troubleshooting)

Если у вас возникли проблемы с запуском, есть универсальный способ диагностики:
Шаг первый: откройте Claude Code в любой директории;
Шаг второй: дайте Claude Code следующую инструкцию:
```
Я установил npm-пакет cc-viewer, но после выполнения ccv он не работает корректно. Посмотри cli.js и findcc.js пакета cc-viewer, определи особенности текущего окружения и адаптируй под локальный способ развёртывания Claude Code. При адаптации старайся ограничить изменения файлом findcc.js.
```
Позволить Claude Code самому проверить ошибки — это эффективнее, чем консультироваться с кем-либо или читать документацию!

После выполнения инструкции файл findcc.js будет обновлён. Если ваш проект часто требует локального развёртывания или вы работаете с форком и регулярно решаете проблемы установки — сохраните этот файл, и в следующий раз просто скопируйте его. На данный момент многие проекты и компании используют Claude Code не на Mac, а на серверном хостинге, поэтому автор вынес findcc.js в отдельный файл для удобства отслеживания обновлений исходного кода cc-viewer.

### Удаление

```bash
ccv --uninstall
```

### Проверка версии

```bash
ccv -v
```

## Возможности

### Мониторинг запросов (режим исходного текста)
<img width="1500" height="720" alt="image" src="https://github.com/user-attachments/assets/519dd496-68bd-4e76-84d7-2a3d14ae3f61" />

- Перехват в реальном времени всех API-запросов Claude Code — именно исходный текст, а не урезанные логи (это важно!!!)
- Автоматическое распознавание и маркировка запросов Main Agent и Sub Agent (подтипы: Plan, Search, Bash)
- Запросы MainAgent поддерживают Body Diff JSON — сворачиваемое отображение различий с предыдущим запросом MainAgent (показываются только изменённые/добавленные поля)
- Встроенная статистика токенов для каждого запроса (входные/выходные токены, создание/чтение кэша, процент попаданий)
- Совместимость с Claude Code Router (CCR) и другими прокси-сценариями — запросы сопоставляются по шаблону пути API

### Режим диалога

Нажмите кнопку «Режим диалога» в правом верхнем углу, чтобы преобразовать полную историю диалогов Main Agent в интерфейс чата:
<img width="1500" height="730" alt="image" src="https://github.com/user-attachments/assets/c973f142-748b-403f-b2b7-31a5d81e33e6" />

- Отображение Agent Team пока не поддерживается
- Сообщения пользователя выровнены вправо (синие пузыри), ответы Main Agent — влево (тёмные пузыри)
- Блоки `thinking` свёрнуты по умолчанию, отрисовываются в Markdown, раскрываются по клику; поддерживается перевод одним нажатием (функция пока нестабильна)
- Сообщения с выбором пользователя (AskUserQuestion) отображаются в формате вопрос-ответ
- Двусторонняя синхронизация режимов: при переключении в режим диалога автоматически выполняется переход к диалогу выбранного запроса; при возврате в режим исходного текста — к выбранному запросу
- Панель настроек: можно переключать состояние сворачивания по умолчанию для результатов инструментов и блоков thinking
- Просмотр диалогов на мобильном: в мобильном режиме CLI нажмите кнопку «Просмотр диалогов» в верхней панели, чтобы открыть панель с историей диалогов в режиме только для чтения

### Управление логами

Через выпадающее меню CC-Viewer в верхнем левом углу:
<img width="1200" height="672" alt="image" src="https://github.com/user-attachments/assets/8cf24f5b-9450-4790-b781-0cd074cd3b39" />

- Импорт локальных логов: просмотр файлов истории логов, сгруппированных по проектам, открытие в новом окне
- Загрузить локальный JSONL-файл: выбор локального файла `.jsonl` для загрузки (поддерживается до 500 МБ)
- Сохранить текущий лог: скачать текущий файл лога мониторинга в формате JSONL

### Автоматическое обновление

CC-Viewer автоматически проверяет обновления при запуске (не чаще одного раза в 4 часа). В пределах одной мажорной версии (например, 1.x.x → 1.y.z) обновления применяются автоматически и вступают в силу при следующем запуске. При смене мажорной версии отображается только уведомление.

Автоматическое обновление следует глобальной конфигурации Claude Code в `~/.claude/settings.json`. Если в Claude Code автоматические обновления отключены (`autoUpdates: false`), CC-Viewer также пропустит автоматическое обновление.
