# CC-Viewer

Система моніторингу запитів Claude Code — перехоплює та візуалізує в реальному часі всі API-запити й відповіді Claude Code (оригінальний текст, без скорочень). Допомагає розробникам відстежувати свій контекст для зручного перегляду та діагностики проблем у процесі Vibe Coding.
Остання версія CC-Viewer також пропонує рішення для серверного веб-розгортання та інструменти для мобільної розробки. Запрошуємо використовувати у своїх проєктах — у майбутньому буде доступно більше можливостей та підтримка хмарного розгортання.

## Використання

### Встановлення

```bash
npm install -g cc-viewer --registry=https://registry.npmjs.org
```

### Режим моніторингу (у цьому режимі запуск claude або claude --dangerously-skip-permissions автоматично створює фоновий процес для запису запитів)

```bash
ccv
```

### Режим програмування

== claude

```bash
ccv -c
```

== claude --dangerously-skip-permissions

```bash
ccv -d
```

Після запуску режиму програмування автоматично відкриється веб-сторінка.

Ви можете використовувати Claude безпосередньо на веб-сторінці, одночасно переглядаючи повні тіла запитів та зміни коду.

І що ще привабливіше — ви навіть можете програмувати з мобільного пристрою!

Ця команда автоматично визначає спосіб встановлення локального Claude Code (NPM або Native Install) та адаптується відповідно.

- **NPM-встановлення**: автоматично впроваджує скрипт перехоплення в `cli.js` Claude Code.
- **Native Install**: автоматично виявляє бінарний файл `claude`, налаштовує локальний прозорий проксі та встановлює Zsh Shell Hook для автоматичного перенаправлення трафіку.
- Рекомендується використовувати Claude Code, встановлений через NPM.

### Перевизначення конфігурації (Configuration Override)

Якщо вам потрібно використовувати власну кінцеву точку API (наприклад, корпоративний проксі), просто налаштуйте її у `~/.claude/settings.json` або встановіть змінну середовища `ANTHROPIC_BASE_URL`. `ccv` автоматично розпізнає та коректно перенаправить запити.

### Тихий режим (Silent Mode)

За замовчуванням `ccv` працює у тихому режимі під час обгортання `claude`, забезпечуючи чистий вивід у терміналі, ідентичний нативному досвіду. Усі логи записуються у фоновому режимі та доступні за адресою `http://localhost:7008`.

Після налаштування просто використовуйте команду `claude` як зазвичай. Відвідайте `http://localhost:7008` для перегляду інтерфейсу моніторингу.

### Усунення несправностей (Troubleshooting)

Якщо у вас виникли проблеми із запуском, є універсальний спосіб діагностики:
Крок перший: відкрийте Claude Code у будь-якому каталозі;
Крок другий: дайте Claude Code таку інструкцію:
```
我已经安装了cc-viewer这个npm包，但是执行ccv以后仍然无法有效运行。查看cc-viewer的cli.js 和 findcc.js，根据具体的环境，适配本地的claude code的部署方式。适配的时候修改范围尽量约束在findcc.js中。
```
Дозволити Claude Code самостійно перевірити помилки — це ефективніше, ніж консультуватися з будь-ким або читати будь-яку документацію!

Після виконання цієї інструкції буде оновле��о findcc.js. Якщо ваш проєкт часто потребує локального розгортання або форкнутий код регулярно стикається з проблемами встановлення — збережіть цей файл. Наступного разу просто скопіюйте його. На даному етапі багато проєктів і компаній використовують Claude Code не на Mac, а на серверному хостингу, тому автор виділив findcc.js в окремий файл для зручного відстеження оновлень вихідного коду cc-viewer.

### Видалення

```bash
ccv --uninstall
```

### Перевірка версії

```bash
ccv -v
```

## Можливості

### Моніторинг запитів (режим оригіналу)
<img width="1500" height="720" alt="image" src="https://github.com/user-attachments/assets/519dd496-68bd-4e76-84d7-2a3d14ae3f61" />

- Перехоплення в реальному часі всіх API-запитів Claude Code — гарантовано оригінальний текст, а не скорочені логи (це дуже важливо!!!)
- Автоматичне розпізнавання та маркування запитів Main Agent і Sub Agent (підтипи: Plan, Search, Bash)
- Запити MainAgent підтримують Body Diff JSON — згорнуте відображення відмінностей від попереднього запиту MainAgent (показуються лише змінені/нові поля)
- Вбудована статистика токенів для кожного запиту (вхідні/вихідні токени, створення/читання кешу, відсоток влучень)
- Сумісність із Claude Code Router (CCR) та іншими проксі-сценаріями — резервне зіставлення запитів за шаблоном шляху API

### Режим діалогу

Натисніть кнопку «Режим діалогу» у верхньому правому куті, щоб перетворити повну історію діалогів Main Agent на інтерфейс чату:
<img width="1500" height="730" alt="image" src="https://github.com/user-attachments/assets/c973f142-748b-403f-b2b7-31a5d81e33e6" />

- Відображення Agent Team наразі не підтримується
- Повідомлення користувача вирівняні праворуч (сині бульбашки), відповіді Main Agent — ліворуч (темні бульбашки)
- Блоки `thinking` згорнуті за замовчуванням, відображаються у Markdown; натисніть, щоб розгорнути та переглянути хід думок; підтримується переклад одним натисканням (функція ще нестабільна)
- Повідомлення з вибором користувача (AskUserQuestion) відображаються у форматі запитання-відповідь
- Двостороння синхронізація режимів: при переключенні на режим діалогу автоматично переходить до діалогу, що відповідає вибраному запиту; при поверненні до режиму оригіналу — автоматично переходить до вибраного запиту
- Панель налаштувань: можна перемикати стан згортання за замовчуванням для результатів інструментів та блоків thinking
- Перегляд діалогів на мобільному: у режимі CLI на мобільному натисніть кнопку «Перегляд діалогу» у верхній панелі, щоб відкрити панель з повною історією діалогів лише для читання

### Автоматичне оновлення

CC-Viewer автоматично перевіряє оновлення при запуску (не частіше одного разу на 4 години). В межах однієї основної версії (наприклад, 1.x.x → 1.y.z) оновлення застосовуються автоматично і набувають чинності при наступному запуску. При зміні основної версії відображається лише сповіщення.

Автоматичне оновлення слідує глобальним налаштуванням Claude Code `~/.claude/settings.json`. Якщо в Claude Code вимкнено автоматичне оновлення (`autoUpdates: false`), CC-Viewer також пропустить автоматичне оновлення.
